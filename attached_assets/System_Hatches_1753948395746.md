## üìñ Introduction

### What are System Hatches?

**System Hatches** are specialized AI teammates that operate globally across the Hatchin platform with elevated privileges and coordination capabilities. Unlike regular Team or Project Hatches, System Hatches can traverse memory layers, coordinate between teams, and facilitate cross-functional collaboration throughout the entire Hatchin ecosystem.

System Hatches serve as the **intelligent coordination layer** that makes Hatchin feel like a cohesive AI workspace rather than isolated chatbots.

### Why They Exist

System Hatches fulfill three critical functions in Hatchin:

1. **üéØ User Onboarding & Guidance**: Guide new users through idea development, team building, and project setup with personalized assistance
2. **üîÑ Cross-Team Coordination**: Facilitate communication, resolve dependencies, and coordinate work between different teams and functional areas
3. **üß† Memory Management**: Access and coordinate information across Project, Team, and Individual Hatch memory layers for seamless collaboration

### How They're Different from Regular Hatches

| **Capability** | **Regular Hatch** | **System Hatch** |
|---------------|-------------------|-------------------|
| **Memory Access** | Own layer + read-only inheritance | Cross-layer read + selective write privileges |
| **Team Boundaries** | Restricted to assigned team | Can coordinate across all teams and projects |
| **Chat Participation** | Team/Project context only | Can appear in any relevant conversation |
| **User Guidance** | Role-specific expertise only | Platform-wide guidance and workflow coordination |
| **Onboarding Role** | None | Primary facilitators of user onboarding flows |
| **Creation Method** | User creates via templates | System-generated for specific coordination needs |
| **Visibility** | Appears in team/project sidebar | Special system section or contextual appearance |
